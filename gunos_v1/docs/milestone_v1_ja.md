# GUNOS V1 マイルストーンノート

本ドキュメントは、GUNOS V1 の最初の主要マイルストーン完了を記録するものです。GUNOS V1 は、既存の GUNO V6 のコアエンジンとデータ基盤をベースにした、プレイ可能なマルチシティプラットフォーム・マイルストーンに到達しました。`gunos_v1/` 配下に独自のアプリケーション層を持つことで、単一都市での実験的な開発から、統一されたマルチシティのプレイアブルなランタイムへのプラットフォームレベルの移行を実現しました。

### 1. マイルストーンの概要

GUNOS V1 は現在、機能的にプレイ可能な状態です。このプラットフォームは、単一のアプリケーションシェル内で複数の現実の都市をサポートしています。マップは主要なゲームプレイ領域へと昇格し、テキスト主体のログから、SVGベースのリッチなネットワーク成長の視覚化へと置き換わりました。ゲームの結果を明確に説明するための、包括的なスコアと結果 UX も実装されています。このマイルストーンにより、GUNOS を単なるプロトタイプではなく、スケーラブルなプラットフォームとして扱うための基盤が整いました。

| 領域 | ステータス |
|------|--------|
| マルチシティ構造 | 完了 |
| メインランタイム | 完了 |
| マップ中心のゲームプレイ | 完了 |
| スコア/結果 UX | 完了 |
| オンライン | スモークテスト済 |
| 特殊システム | 延期 |

### 2. 完了したフェーズ

V1 への移行は、以下の6つのフェーズを通じて達成されました：

- **Phase 1 — スキャフォールド構築**: `gunos_v1/` のディレクトリ構造を確立し、プラットフォームのエントリーポイントを作成。既存の `guno_v6` の都市プロファイルを解決できるようシティローダーを適合させました。
- **Phase 2 — メインプレイレイアウト**: アプリケーションシェルを、専用のヘッダーバー、マップパネル、下部コントロールパネルを備えた、プロダクト品質のマップ中心レイアウトに再構築しました。
- **Phase 3 — プレイエンジン統合**: `guno_v6` のプレイエンジンとデッキ生成器を新しいシェルに接続し、START/RESET 機能、ターン進行、ライブハンド表示を可能にしました。
- **Phase 4 — マップの視覚化**: プレースホルダーのマップビューを SVG ベースの地理的レンダラーに置き換え、駅ノード、路線エッジ、リアルタイムのプレイヤー所有権オーバーレイを視覚化しました。
- **Phase 5 — スコア/結果 UX**: 最終スコアリングエンジンを統合し、ゲームプレイ中のライブスコア追跡を導入。プレイヤーが「なぜ勝ったのか」を説明する包括的な GAME OVER 結果パネルを構築しました。
- **Phase 6 — ランタイムの明瞭性とクロスシティの洗練**: 明確なゲーム状態バッジ、ターンカウンター、より強いネットワーク所有権のグロー効果、クロスシティの特性テキストを追加し、UI を洗練させてプレイアブルな体験を完成させました。

### 3. サポートされる都市

GUNOS V1 は現在、4つの現実の都市パッケージをサポートしています。すべての都市は同じエンジンを共有しながら、それぞれ異なるネットワークトポロジーを提供します。

- **Tokyo** (`data_ready: true`): メインランタイムは完全に動作します。激しく争われるハブ駅を持つ、高密度のベースラインネットワークが特徴です。
- **Osaka** (`data_ready: true`): メインランタイムは完全に動作します。コンパクトでバランスの取れたネットワークが特徴です。
- **London** (`data_ready: true`): メインランタイムは完全に動作します。乗り換えの多いメトロコアが特徴です。（スコアスキーマの違いも正常に正規化済みです）。
- **NYC** (`data_ready: true`): メインランタイムは完全に動作します。長距離のマンハッタンハブシステムが特徴です。

### 4. 現在の機能

GUNOS V1 は現在、以下の操作を実行できます：
- 都市を認識したスタートアップとブートシーケンス
- UI セレクターまたは URL パラメータ（`?city=`）によるシームレスな都市切り替え
- データを複製することなく、特定の都市パッケージとデータセットをロード
- 実際の駅グラフに基づいた、都市固有のカードデッキの生成
- ゲームセッションの開始とリセット
- ターンベースのランタイムの実行（PLAY 1 / AUTO ×20）
- マップ上でのリアルタイムなプレイヤー所有権とネットワーク成長の表示
- ライブスコアと詳細なゲーム終了結果の表示
- 複数の異なる都市トポロジーにわたる一貫した実行
- 基本的なオンライン層（Supabase Realtime）への接続（初期スモークテスト合格済み）

### 5. アーキテクチャ上の意義

アーキテクチャの観点から、このマイルストーンは関心の分離の有効性を証明しています：
- `guno_v6/` は、信頼できる唯一の情報源（Source of Truth）としてのエンジンおよびデータ環境であり続けます。保護されており、変更されていません。
- `gunos_v1/` は、プラットフォーム向けのアプリケーション層となり、エンジンをライブラリとして消費します。
- 都市パッケージは完全に再利用可能であり、UI 層から独立しています。
- まったく同じプレイエンジンが、注入されるデータコンテキストを切り替えるだけで、複数の現実の都市を動的に駆動します。
- マップがゲームプレイの中心となり、複雑な地理的グラフデータがブラウザ上で動的にレンダリングできることが証明されました。

### 6. 意図的に未完了としている項目

開発の勢いを維持するため、いくつかの機能は意図的に延期されており、V1 マイルストーンの一部とは見なされません：
- 完全なオンラインの安定化とマルチプレイヤーのマッチメイキング UI
- 特殊カードシステム（イベントカード、アクションカードなど）
- リプレイシステムや詳細なターン履歴ログ
- 大規模な演出の洗練（アニメーション、効果音、パーティクルシステム）
- 最終的なローンチパッケージングとデプロイメントパイプライン
- 個々の都市ごとの深いゲームプレイバランス調整とルートサイズのチューニング
- 高度なルート/ネットワーク説明 UI（どの路線がルートボーナスに貢献したかを正確にハイライトするなど）

これらの項目は、忘れられているわけではなく、コアプラットフォームの安定性を優先するために意図的に延期されています。

### 7. 既知の制限事項 / 既知の問題

V1 ビルドの現在の現実的な制限事項は以下の通りです：
- ルートとネットワークのライブ可視性は、ノード密度によって都市間でまだ不均一な場合があります。
- ルートボーナスなどの一部のスコア要素は、路線の長い都市の方が他の都市よりも明確に表れる場合があります。
- ロンドンと NYC のマップは、密集エリアでの駅の重なりにより、可読性の向上やチューニングがまだ必要な場合があります。
- オンライン層はスモークテストのみが行われており、エッジケースや切断に対して完全に強化されているわけではありません。
- 都市比較 UI は、シンプルなセレクタードロップダウン/ボタン行に限定されています。

### 8. V1.1 に含まれるもの

次のイテレーション（V1.1）に向けた前向きなバックログには以下が含まれます：
- オンラインの安定化と堅牢なルーム管理
- ルート/ネットワーク可視性の洗練（ホバーによる特定ルートのハイライトなど）
- リプレイとターン履歴のタイムライン UI
- 強化された都市比較 UI（統計、リーダーボード）
- ロンドンと NYC 固有の視覚的チューニング（情報の整理）
- 追加の QoL（Quality of Life）改善（ツールチップ、キーボードショートカット）
- より良い結果説明（マップ上でのスコア源の視覚的内訳）

### 9. 推奨されるマイルストーン・ステートメント

GUNOS V1 は、検証済みの GUNO V6 基盤の上に構築され、反復的な V1.x の洗練の準備が整った、最初のプレイ可能なマルチシティプラットフォーム・マイルストーンとして扱われるべきです。
