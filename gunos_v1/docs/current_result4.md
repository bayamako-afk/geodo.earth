# Completion Report
GUNOS V1.1 Task 04 — London / NYC Readability Tuning (Map Label Filter)

## Summary
London / NYC などの大規模ネットワークにおいて、駅名ラベルが密集して可読性が低下する問題を解決するため、`hub_degree` に基づくラベル表示フィルターを実装しました。各都市の実際のネットワーク特性（ノード数と最大 degree）に合わせて閾値を動的に設定し、中心部の密集を解消しつつ、重要なハブ駅やプレイヤー所有駅のラベルは常に表示されるように改善しました。非表示となったラベルも、ノードにホバーすることで確認可能です。

## Files Changed
- `src/ui/map_canvas.js`
  - 都市規模に応じた `hub_degree` の動的閾値判定関数 `_labelThreshold()` を追加
  - ラベル描画条件に閾値を適用し、条件を満たさない場合は `.map-node-label--hidden` クラスを付与するように修正
- `index.html`
  - `.map-node-label--hidden` クラスとホバー時の表示制御（CSS）を追加
  - `<g class="map-node">` への `pointer-events: all` を追加し、ホバー判定を有効化
  - キャッシュバスターを `v=10` に更新

## What Was Done
1. **都市別ネットワーク特性の調査**
   - 各都市のノード数と `hub_degree` の分布を調査しました。
   - London（162ノード）は最大 degree が 4、NYC（152ノード）は最大 degree が 6 であり、単純な固定閾値では都市間のバランスが取れないことが判明しました。
2. **動的閾値フィルターの実装**
   - 調査結果に基づき、以下の閾値を設定しました：
     - **London / NYC**（ノード数100以上）：`hub_degree >= 3` のみ表示
     - **Tokyo / Osaka**（ノード数100未満）：`hub_degree >= 2` のみ表示（従来通り）
3. **ホバー時のラベル表示機能（CSS）**
   - 閾値未満の駅ラベルは初期状態では非表示（`opacity: 0`）となりますが、ノードにマウスホバーした際のみ表示（`opacity: 0.85`）されるように CSS で制御を追加しました。
   - これにより「全体の見やすさ」と「詳細情報の確認」を両立させています。
4. **ゲームプレイ上重要なラベルの保護**
   - プレイヤーが所有している駅（`isOwned`）や、現在プレイ中のカードの駅（`isCurrent`）は、閾値に関わらず常に表示されるようにしています。

## Test Result
全4都市での動作テストを実施し、以下の結果を確認しました。

| 都市 | ノード数 | 適用閾値 | ラベル表示状況 |
|------|---------|---------|--------------|
| NYC | 152 | degree ≥ 3 | マンハッタン中心部の過密が解消。主要ハブ（Times Sq 等）は表示。 |
| London | 162 | degree ≥ 3 | King's Cross 等の中心部の重なりが激減。外周部は適度に表示。 |
| Tokyo | 86 | degree ≥ 2 | 従来通り、主要な乗換駅のラベルが適度に表示される状態を維持。 |
| Osaka | 85 | degree ≥ 2 | 従来通り、見やすい状態を維持。 |

- **ホバー挙動**: 全都市において、非表示のノードにマウスを乗せるとラベルが浮かび上がることを確認しました。
- **コンソールエラー**: 全都市でオートプレイを実行し、コンソールエラーが 0 件であることを確認しました。
