# GUNOS V1.4 Task 02 Completion Report

## 概要
GUNOS V1.4 Task 02「Next Candidate Indicators」の実装が完了しました。このタスクでは、プレイヤーが「次にどの駅を狙うべきか」を視覚的に提示し、ゲームの戦略性を高めるための新しいUIパネルをマップオーバーレイに追加しました。

## 実装内容の詳細

### 1. 候補駅選定ロジックの実装
`candidate_indicator.js` モジュールを新規作成し、現在のプレイヤーの手札（未取得の駅）の中から、現在のゲーム状況に基づいて最適な候補駅を1〜3件選定するロジックを実装しました。

評価基準として、以下の要素を考慮しています。
- **ハブ価値**: 駅の基礎スコアが都市の最大スコアに対してどの程度の割合を占めるかを評価し、`Hub★`、`Hub+`、`Hub` のタグを付与します。
- **ルート構築**: プレイヤーがすでに取得している路線の進行度（50%以上で `Chain`、20%以上で `Connect`）を評価し、路線の延長や接続の価値を判定します。
- **ルートボーナス**: すでにルートボーナスが発生している路線をさらに強化できる場合に `Route+` タグを付与します。
- **リードギャップ**: 現在のプレイヤーが負けており、かつトップとのスコア差が5ポイント以上ある場合、逆転の足がかりとして `Lead+` タグを付与します。
- **純粋なスコア価値**: 上記の条件に当てはまらない場合でも、駅自体のスコアが高い場合に `Score★` や `Score` タグを付与します。

これらの評価に基づいて候補駅をスコアリングし、上位3件を抽出して表示します。

### 2. UIコンポーネントの追加
`index.html` に `#candidate-panel` のCSSおよびHTML構造を追加しました。このパネルは `#map-overlay-info` 内の左下（Task 01で実装した Station Hint Panel のすぐ上）に配置されます。

パネルのデザインは、既存のHUD（ヘッドアップディスプレイ）と調和するよう、半透明のダークな背景とブラー効果（`backdrop-filter: blur(10px)`）を採用しています。各候補駅には、駅名、評価タグ（最大2つ）、およびその駅を推奨する理由を簡潔に示す1行のテキストが表示されます。

### 3. レスポンシブ対応
デバイスの画面サイズや向きに応じて、パネルの表示が最適化されるようメディアクエリを実装しました。
- **デスクトップ**: 最大3件の候補駅を表示し、タグと理由テキストを完全に表示します。
- **モバイル縦（Portrait）**: パネルの幅を縮小し、最大2件の候補駅を表示します。スペースの都合上、理由テキストは非表示になります。
- **モバイル横（Landscape）**: パネルの幅と高さを最小限に抑え、最大2件の候補駅を表示します。フォントサイズを縮小し、理由テキストは非表示になります。

### 4. アプリケーションへの統合
`main.js` において、`initCandidateIndicator`、`updateCandidateIndicator`、および `resetCandidateIndicator` をインポートし、ゲームのライフサイクルに合わせて呼び出すように統合しました。
- **初期化**: `boot` 関数内で、データセット（`station_metrics`、`station_lines`、`lines_master`）を渡してパネルを初期化します。
- **更新**: `_updateAllPanels` 関数内で、毎ターンの状態変化に応じてパネルの内容を更新し、フェードインアニメーションで表示します。
- **リセット**: `_handleReset` 関数内でパネルを非表示にし、状態をリセットします。

## テスト結果
ローカルサーバーを起動し、デスクトップおよびモバイルのシミュレーション環境でテストを実施しました。
- ゲーム開始直後から、P1のターンにおいて適切な候補駅が3件表示されることを確認しました。
- `AUTO ×20` を実行し、ゲーム進行に伴って候補駅や付与されるタグ（`Hub+`、`Lead+` など）が動的に変化することを確認しました。
- ゲーム終了（GAME OVER）後も、最終ターンの候補パネルが表示されたまま維持されることを確認しました。
- `RESET` ボタン押下時に、パネルが正しく非表示（`opacity: 0`）になることを確認しました。

この機能により、プレイヤーは現在の局面における戦略的な選択肢を直感的に理解できるようになりました。次のステップとして、V1.4 Task 03「Hub / Route+ meaning badges」の実装に進むことで、ゲームのルールやスコアの仕組みをさらに分かりやすく伝えることが期待されます。
