# 従業員・通信機器・電話番号 統合管理システム (SPFx Web Part)

SharePoint Framework (SPFx) 1.20 / React 17 で構築する、従業員マスタ（社員台帳）と連動した通信機器（SIM・端末）および電話番号（スマホ・Teams外線）の統合管理ウェブパートです。

## 1. システム概要と目的

これまでExcelで管理していた「社員台帳」をSharePointリスト化し、その従業員情報に対して「どの端末（IMEI）」「どのSIM（ICCID）」「どの電話番号（スマホ/Teams）」が割り当てられているかを一元管理します。

これにより、入社時の機器貸与から退社時の回収までを正確に追跡し、遊休資産（在庫）や解約漏れを防ぐことを目的とします。

## 2. SharePointリスト構成

本システムは以下の5つのSharePointリストを連携させて動作します。
ウェブパートの「リスト初期化」ボタンから自動作成が可能です。

### 2.1. `Employee_Master` (従業員マスタ / 社員台帳)
社員の基本情報を管理します。

| 内部列名 | 表示名 | データ型 | 説明 |
| :--- | :--- | :--- | :--- |
| `Title` | 従業員番号 | 1行テキスト | 独自採番の社員番号（例: EMP-001） |
| `EmployeeName` | 氏名 | 1行テキスト | 社員名 |
| `Department` | 部署 | 1行テキスト | 所属部署 |
| `JobTitle` | 役職 | 1行テキスト | 役職名 |
| `Status` | 在籍状況 | 選択肢 | `在籍`, `休職`, `退職` |
| `JoinDate` | 入社日 | 日付と時刻 | |
| `LeaveDate` | 退社日 | 日付と時刻 | |
| `Remarks` | 備考 | 複数行テキスト | |

### 2.2. `PhoneNumber_Master` (電話番号マスタ)
スマホの番号やTeams外線番号などの「番号」そのものを資産として管理します。

| 内部列名 | 表示名 | データ型 | 説明 |
| :--- | :--- | :--- | :--- |
| `Title` | 電話番号 | 1行テキスト | ハイフンありの電話番号 |
| `NumberType` | 番号種別 | 選択肢 | `スマホ(SIM紐付)`, `Teams外線`, `固定電話`, `その他` |
| `Carrier` | キャリア/プロバイダ | 選択肢 | `docomo`, `au`, `SoftBank`, `Microsoft`, `その他` |
| `Status` | 状態 | 選択肢 | `利用中`, `空き(未割当)`, `解約済` |
| `Remarks` | 備考 | 複数行テキスト | |

### 2.3. `SIM_Master` (SIMマスタ)
物理的なSIMカードを管理します。

| 内部列名 | 表示名 | データ型 | 説明 |
| :--- | :--- | :--- | :--- |
| `Title` | SIM識別番号 | 1行テキスト | ICCIDや製造番号（例: AX0605980760120） |
| `PhoneNumberId` | 電話番号参照 | 参照 | `PhoneNumber_Master`へのルックアップ（音声SIMの場合） |
| `Carrier` | 通信キャリア | 選択肢 | `docomo`, `au`, `SoftBank`, `その他` |
| `PlanName` | 契約プラン名 | 1行テキスト | 例: ギガホ、データ通信専用など |
| `SimType` | SIM種別 | 選択肢 | `データSIM`, `音声通話SIM(携帯)` |
| `Status` | 状態 | 選択肢 | `利用中`, `在庫(未割当)`, `解約済`, `紛失` |
| `MonthlyCost` | 月額費用 | 数値 | 基本料金（参考値） |
| `Remarks` | 備考 | 複数行テキスト | |

### 2.4. `Device_Master` (端末マスタ)
スマートフォンやモバイルルーターなどの物理端末を管理します。

| 内部列名 | 表示名 | データ型 | 説明 |
| :--- | :--- | :--- | :--- |
| `Title` | IMEI | 1行テキスト | 端末識別番号（15桁） |
| `SerialNumber` | シリアル番号(S/N) | 1行テキスト | メーカー製造番号 |
| `DeviceModel` | 機種名 | 1行テキスト | 例: iPhone 13, iPad Pro, モバイルルーター |
| `DeviceType` | 端末種別 | 選択肢 | `スマートフォン`, `タブレット`, `ルーター`, `その他` |
| `Status` | 状態 | 選択肢 | `利用中`, `在庫`, `故障`, `廃棄` |
| `PurchaseDate` | 購入日 | 日付と時刻 | |
| `Remarks` | 備考 | 複数行テキスト | |

### 2.5. `Asset_Allocation` (貸与・割当管理)
「誰に」「何を」貸与しているかの履歴と現在の状態を管理します。

| 内部列名 | 表示名 | データ型 | 説明 |
| :--- | :--- | :--- | :--- |
| `Title` | 割当ID | 1行テキスト | 自動採番 |
| `EmployeeId` | 従業員参照 | 参照 | `Employee_Master`へのルックアップ |
| `AllocationType`| 割当対象種別 | 選択肢 | `SIM+端末セット`, `端末のみ`, `SIMのみ`, `Teams外線のみ` |
| `SimId` | SIM参照 | 参照 | 物理SIMを渡している場合 |
| `DeviceId` | 端末参照 | 参照 | 物理端末を渡している場合 |
| `PhoneNumberId` | 電話番号参照 | 参照 | Teams外線など番号だけを割り当てている場合 |
| `StartDate` | 貸与開始日 | 日付と時刻 | |
| `EndDate` | 貸与終了日 | 日付と時刻 | 返却・変更日 |
| `IsCurrent` | 現在利用中 | はい/いいえ | 最新の割当状態かどうか |
| `Notes` | 特記事項 | 複数行テキスト | |

## 3. UI/UX 設計 (SPFx Web Part)

UIはFluent UIを使用し、余白を最小限に抑えたコンパクトで視認性の高い一覧表示を基本とします。
1つの画面内で以下の4つのタブを切り替えて操作します。

### 3.1. タブ構成

1. **社員台帳（従業員・割当状況）**
   - 従業員の一覧と、現在その従業員に割り当てられている「端末」「電話番号」「Teams外線」を結合して一覧表示。
   - Excelの社員台帳の代替となるメイン画面。
   - 新規社員登録、割当（貸与）の追加・変更・返却操作。
2. **在庫・資産一覧（端末 / SIM / 番号）**
   - 端末、SIM、電話番号それぞれの「在庫（未割当）」や「利用中」の状態を一覧表示。
   - 端末やSIMの新規購入・契約時の登録画面。
3. **ダッシュボード**
   - 部署別の回線利用数、端末の在庫数、Teams外線の割当状況などのサマリー表示（コンパクトなカード型UI）。
4. **解約・退職者・履歴**
   - 退職した社員、解約済みのSIM/番号、廃棄された端末、過去の割当履歴。

### 3.2. 操作アクション

- **社員入社時**: 従業員マスタに新規登録し、在庫から端末・SIM・Teams外線を選択して一括割当。
- **機器変更（故障など）**: 現在の割当を「返却（終了）」とし、新しい端末を「新規割当」する履歴管理。
- **社員退社時**: 従業員ステータスを「退職」にし、現在割り当てられている全ての資産を「在庫（空き）」に戻す一括処理。

## 4. 開発・実装方針

1. **プロジェクト構成**: `@microsoft/generator-sharepoint` (v1.20.0) / React 17 / Fluent UI 8.x
2. **データアクセス**: `@pnp/sp` (PnPJS) を使用し、ODataクエリで複数リストを結合・高速取得。
3. **自動セットアップ**: 初回起動時に5つのリストとフィールドを自動作成する機能を実装。
4. **エクスポート機能**: SheetJS (`xlsx`) を使用し、現在表示されている一覧（社員台帳や在庫リスト）をExcel出力。

## 5. 権限設計

- **システム管理者（情報システム部等）**: サイトの「編集」権限以上。すべてのマスタ登録・編集・削除、割当操作が可能。
- **一般閲覧者（各部署のマネージャー等）**: サイトの「閲覧」権限。社員台帳や割当状況の閲覧と検索、Excelエクスポートのみ可能。

## 6. ビルド・デプロイ手順

### 前提条件
- Node.js 18.x
- SharePoint Framework (SPFx) v1.20.0 開発環境

### 依存関係のインストール
```bash
npm install
```

### ビルドとパッケージング
```bash
# クリーンアップ
gulp clean

# バンドル（プロダクション用）
gulp bundle --ship

# sppkgファイルの作成
gulp package-solution --ship
```

### SharePointへのデプロイ
1. 作成された `sharepoint/solution/device-manager.sppkg` をSharePoint管理センターの「アプリカタログ」にアップロードします。
2. 利用したいSharePointサイトで「アプリの追加」から「通信機器・社員台帳 管理システム」を追加します。
3. サイトのページにウェブパートを配置します。
4. （初回のみ）ウェブパート右上の「リスト初期化」ボタンをクリックして、必要な5つのSharePointリストを自動作成します。
